# TikTokライバーマネジメントシステム 要件定義書

## 1. プロジェクト概要
TikTokライバー管理者の業務効率化および、所属ライバーのモチベーション向上、自社ブランディングを目的としたシステム群を構築する。

## 2. 機能要件

### 2.1 自社コーポレートサイト
- **会社概要表示:** 組織のビジョン、所属ライバー紹介等の掲載。
- **Tikfinity連携ガチャ機能:** 
  - Tikfinity のポイントAPI（または連携機能）を利用。
  - サイト上でポイントを消費し、景品や演出が当たるガチャシミュレーターの実装。

### 2.2 活動状況データ自動抽出ツール
- **データソース:** `https://live-backstage.tiktok.com/` (TikTok Live Backstage)
- **自動化処理:** 
  - ログインおよびデータのエクスポート処理の自動化（スクレイピングまたはRPA的アプローチ）。
  - 抽出したデータを Excel 形式から Google スプレッドシートへ自動同期。

### 2.3 ライバー向けランキングサイト
- **目的:** Backstage権限を持たないライバーへの情報公開。
- **表示内容:** 抽出データに基づくランキング（ギフト数、視聴者数、配信時間等）。
- **セキュリティ:** 外部に漏洩しないよう、簡易認証（パスワード保護等）の検討。

## 3. 非機能要件
- **運用形態:** Docker Compose を用いたコンテナベースの運用。
- **拡張性:** 将来的なライバー数の増加に耐えうるデータ保持構造。
- **保守性:** TikTok側のサイト仕様変更に迅速に対応できる構成。
- **実行サイクル:** 活動データの抽出は週次（毎週月曜日 26:00）で実施。

## 4. 技術的検討事項（懸念点）
- **TikTokログインの自動化:** 
  - Live Backstageは二要素認証(2FA)やCAPTCHAが導入されている可能性が高く、完全自動化にはヘッドレスブラウザ（Playwright/Puppeteer）と手動ログイン保持の組み合わせが必要。
- **Tikfinity API:** 
  - ポイント消費をサイト側からトリガーできるか、APIドキュメントの確認が必要。
- **データプライバシー:** 
  - ランキングサイトに表示する情報の粒度（他者の詳細な収益が見えすぎないか等）の調整。

## 5. 構成案（仮）
- **Frontend:** Next.js / React (コーポレート & ランキング)
- **Worker:** Python (Playwright/Selenium)
- **Database:** 不要（Googleスプレッドシートおよびローカルファイルをデータストアとして利用）
- **Infrastructure:** Docker Compose
