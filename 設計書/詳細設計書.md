# TikTokライバーマネジメントシステム 詳細設計書

## 第1章 システム構成・コンテナ設計

### 1.1 コンテナ一覧と役割
Docker Compose を利用して以下の 4 つのコンテナを稼働させる。

| コンテナ名 | サービス名 | イメージ / ベース | 役割 |
| :--- | :--- | :--- | :--- |
| `nginx` | `reverse-proxy` | `nginx:alpine` | SSL終端、パスベースのルーティング（L7） |
| `corporate-site` | `frontend-corp` | `node:22-alpine` | 自社HP、Tikfinity連携（Next.js） |
| `tiktok-ranking` | `frontend-rank` | `node:22-alpine` | ライバー向けランキングサイト（Next.js） |
| `data-fetcher` | `worker-fetcher` | `python:3.13-slim` | Seleniumによるデータ抽出・加工（Python） |

### 1.2 ネットワーク設計
Docker 内部ネットワーク `app-network` (bridge) を構築し、外部公開は `nginx` コンテナのみを経由する。

- **Nginx ルーティングルール:**
  - `http://[Domain]/` → `corporate-site:3000`
  - `http://[Domain]/ranking` → `tiktok-ranking:3000`
  - 静的ファイルのキャッシュ設定、Gzip 圧縮を有効化。

### 1.3 ボリューム設計（データ共有・永続化）
DBを使用しないため、ボリュームマウントによるファイル連携を厳密に定義する。

| ホストパス (相対) | コンテナパス | 関連コンテナ | 用途 |
| :--- | :--- | :--- | :--- |
| `./data/creators` | `/app/data` | `data-fetcher`, `tiktok-ranking` | 抽出済みCSV/Excelデータの格納・参照 |
| `./data-fetcher/profile` | `/app/profile` | `data-fetcher` | Selenium のブラウザプロファイル保存（ログイン維持） |
| `./logs` | `/app/logs` | 全コンテナ | 各種実行ログの集約 |
| `./nginx/conf.d` | `/etc/nginx/conf.d` | `nginx` | Nginx 設定ファイルの流し込み |

### 1.4 Git Submodule 構成
メインリポジトリの配下に各サービスのリポジトリを配置する。

```text
root/
├── .gitmodules
├── compose.yaml
├── nginx/
│   └── conf.d/default.conf
├── services/
│   ├── corporate-site/ (Submodule: 将来の実装)
│   ├── tiktok-ranking/ (Submodule: 将来の実装)
│   ├── data-fetcher/   (Submodule: 実装済み)
│   └── dummy/          # 暫定ダミーサイト用 (Dockerfile, index.html)
├── data/ (Git管理対象外)
└── secrets/            # Git管理対象外
    ├── .env.common     # 共通設定（タイムゾーン等）
    ├── .env.google     # Google API関連（Sheet ID等）
    ├── .env.corporate  # コーポレートサイト専用（Tikfinity等）
    ├── .env.fetcher    # データ抽出Worker専用（TikTok ID/PW等）
    └── service-account.json # Google認証キー
```

### 1.5 環境変数管理 (.env)
ルートディレクトリの `.env` で一括管理し、各コンテナへ必要な項目を渡す。

```bash

# TikTok Credentials
TIKTOK_USERNAME=your_id
TIKTOK_PASSWORD=your_password

# Google Sheets API

GOOGLE_SERVICE_ACCOUNT_JSON=/app/secrets/service-account.json

```

### 2.2 画面一覧
| パス | 画面名 | 役割 |
| :--- | :--- | :--- |
| `/` | トップページ | ビジョン、最新トピックス |
| `/about` | 会社概要 | 沿革、所在地、代表挨拶 |

| `/gacha` | ガチャページ | Tikfinityポイントを利用したガチャシミュレーター |

### 2.3 ガチャ機能詳細
Tikfinityの外部APIと連携し、ポイント消費と景品抽選を行う。

#### 2.3.1 シーケンス
1. **ユーザー:** ガチャ実行ボタンをクリック。
2. **フロントエンド:** API Route `/api/gacha/play` へPOSTリクエスト。
3. **API Route (Server):** 
    - 環境変数のAPIキーを使用して Tikfinity API へポイント消費リクエストを送信。
    - ポイント不足等のエラーがないかチェック。
    - Googleスプレッドシート（Configシート）から最新の景品リストと確率を取得（キャッシュ活用）。
    - サーバーサイドで乱数による景品抽選を実行。
4. **API Route (Server):** 抽選結果（景品ID、名称、演出タイプ）をフロントエンドへ返却。
5. **フロントエンド:** `framer-motion` を使用した演出アニメーションを実行後、結果を表示。

#### 2.3.2 API仕様
**POST `/api/gacha/play`**
- **Request Body:**
  ```json
  { "userId": "string", "gachaId": "string" }
  ```
- **Response (Success):**
  ```json
  {
    "success": true,
    "prize": {
      "id": "p001",
      "name": "オリジナル限定ステッカー",
      "rarity": "SR",
      "animationType": "gold_spark"
    },
    "remainingPoints": 850
  }
  ```

#### 2.3.3 景品マスタ管理 (Google Sheets)
`Config` スプレッドシートの `Prizes` シートをマスターとする。

| ID | 景品名 | 確率(%) | 演出フラグ | 有効期限 |
| :--- | :--- | :--- | :--- | :--- |
| p001 | 限定壁紙 | 70.0 | basic | - |
| p002 | ボイスメッセージ | 25.0 | rare | 2025-12-31 |
| p003 | 個別ビデオレター | 5.0 | super_rare | 2025-06-30 |

### 2.4 Tikfinity API 連携
- **認証:** `Authorization: Bearer ${TIKFINITY_API_TOKEN}`
- **エンドポイント:** 
    - `GET /api/points/${userId}`: 所持ポイント確認
    - `POST /api/points/consume`: ポイント消費実行

### 2.5 問い合わせ機能
- **送信先:** Googleスプレッドシートの `Inquiry` シートへ直接書き込み。
- **通知:** Discord Webhookを使用して、送信内容を管理チャンネルへ即時転送。

## 第3章 tiktok-ranking 詳細設計

### 3.1 フロントエンド構成
- **フレームワーク:** Next.js 15 (App Router)
- **認証:** NextAuth.js (Credentials Provider)
- **主要ライブラリ:**
  - `csv-parse`: 共有ボリュームのCSV解析用
  - `recharts`: 活動推移グラフ表示用
  - `clsx / tailwind-merge`: スタイリング管理

### 3.2 画面一覧
| パス | 画面名 | 役割 |
| :--- | :--- | :--- |
| `/ranking/login` | ログイン画面 | ライバー専用パスワードによる認証 |
| `/ranking` | ダッシュボード | 週間ランキング表示、全体サマリ |
| `/ranking/me` | マイページ | 自身の詳細推移グラフ、過去実績 |

### 3.3 データ連携メカニズム（DBレス・ファイル参照）
`data-fetcher` が出力するCSVファイルをリアルタイムまたは一定間隔で読み込む。

#### 3.3.1 ファイル読み込み処理
サーバーサイド（Server Components）で `fs.readFileSync` を使用して共有ボリューム `/app/data/ranking_data.csv` を直接参照する。

```typescript
// 実装イメージ
import { parse } from 'csv-parse/sync';
import fs from 'fs';

export async function getRankingData() {
  const fileContent = fs.readFileSync('/app/data/ranking_data.csv', 'utf8');
  const records = parse(fileContent, { columns: true, skip_empty_lines: true });
  return records;
}
```

#### 3.3.2 性能確保 (ISR)
ファイルの読み込み負荷を抑えるため、Next.jsの **Incremental Static Regeneration (ISR)** を活用。1時間（3600秒）間隔で再生成を行う設定とする。

### 3.4 認証ロジック
専用データベースを持たないため、Googleスプレッドシートの `Livers` シートを認証マスタとして利用する。

1. **ログイン試行:** ユーザーがライバーIDとパスワードを入力。
2. **バックエンド検証:** 
   - `googleauth` を使用し、スプレッドシートから該当ライバーのレコードを取得。
   - 入力されたパスワードとシート上のパスワードを照合。
3. **セッション発行:** 照合成功時、JWT（JSON Web Token）にライバーIDを保持してセッションを開始。

### 3.5 表示項目とソート
- **ソート項目:** ギフト数（デフォルト）、視聴者数、配信時間。
- **ランキング制限:** 全体の順位はTOP 50まで表示。自身の順位は常に最下部に固定表示、または強調表示する。
- **秘匿性:** 他者の詳細なギフト数（収益に直結する値）は非表示とし、スコア化または順位のみの表示とする調整を行う。

## 第4章 data-fetcher 詳細設計

### 4.1 実行環境・スタック
- **言語:** Python 3.13-slim
- **主要ライブラリ:**
  - `selenium`: ブラウザ自動操作用
  - `webdriver-manager`: ChromeDriverの自動管理
  - `pandas`, `openpyxl`: エクセル・CSVデータの加工・整形用
  - `google-api-python-client`, `google-auth`: Googleスプレッドシート連携用
  - `requests`: Discord通知（Webhook）用

### 4.2 処理フロー詳細
毎週月曜日 26:00 に起動し、以下のフローを自動実行する。

#### 4.2.1 ブラウザ起動とログイン
- Dockerボリュームマウントされた `/app/profile` を `--user-data-dir` に指定して起動。
- ログイン済みセッションの有無を確認。セッションが切れている場合は、Discordへ通知し手動介入を促す（または予め保存した認証情報を再入力）。

#### 4.2.2 データ抽出（スクレイピング）
1. TikTok Live Backstage (`https://live-backstage.tiktok.com/`) へアクセス。
2. サイドメニューから「データ」→「ライブ統計」を選択。
3. 期間フィルタで「過去7日間」を選択。
4. 「エクスポート」ボタンをクリックし、コンテナ内の `/tmp/download` に保存。

#### 4.2.3 データ加工（Pandas）
ダウンロードしたExcel/CSVファイルを `pandas` で読み込み、以下のクレンジングを行う。
- 不要なヘッダー・フッター行の削除。
- `tiktok_id` をキーとして、マスタ情報と突合。
- ランキング算出用のスコア計算（例: ギフト数 × 1.0 + 視聴者数 × 0.5）。

#### 4.2.4 データ配信
1. **共有ボリューム保存:** 
   - 整形後のデータを `/app/data/ranking_data.csv` として上書き保存。
   - 履歴管理用として `/app/data/archive/ranking_YYYYMMDD.csv` を作成。
2. **Googleスプレッドシート同期:**
   - `History` シートの末尾に、今回の集計結果（週次データ）を追記。
   - `Livers` シートの最終アクティブ日などのメタデータを更新。

### 4.3 セキュリティ・安定化設計
- **ヘッドレスモード:** 基本は headless で実行するが、デバッグ時や初回認証時のみ VNC または画面表示可能なモードへ切り替え可能とする。
- **待機処理:** TikTok側の要素読み込み遅延に対応するため、`WebDriverWait` による明示的な待機を徹底する。
- **エラー通知:** 処理の各ステップ（ログイン、抽出、加工、配信）で例外が発生した際、スタックトレースの一部を Discord Webhook で送信。

### 4.4 共有ボリューム構造 (詳細)
```text
/app/data/
├── ranking_data.csv       # ランキングサイト参照用 (最新)
├── archive/               # 過去の集計データ (バックアップ)
│   └── ranking_20250127.csv
└── config/                # 補助的な設定ファイル (必要な場合)
```

### 4.5 実装イメージ（Python）
```python
from selenium import webdriver
from selenium.webdriver.chrome.options import Options

def setup_driver():
    options = Options()
    options.add_argument('--headless')
    options.add_argument('--no-sandbox')
    options.add_argument(f'--user-data-dir=/app/profile')
    # その他、安定化のためのオプション
    driver = webdriver.Chrome(options=options)
    return driver
```
